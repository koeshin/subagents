# Daily Walkthrough Workflow

오늘 진행한 작업을 정리하여 walkthrough 문서를 생성합니다.

## 사용법

```
/daily                   # 전체 프로세스 실행 (수집 → 작성 → 검토)
/daily collect           # 오늘 작업 수집만
/daily write             # 문서 작성만
/daily review            # 검토만
```

## 프로세스

```
1. Collect (daily-collector)
   오늘 작업 내역 수집
        ↓
2. Write (daily-writer)
   walkthrough 문서 작성
        ↓
3. Review (daily-reviewer)
   문서 검토 및 개선
```

---

## 단계 1: Collect

오늘 진행한 작업을 파악합니다.

**수집 대상:**
- Git 커밋 내역
- 변경된 파일
- 생성/삭제된 파일
- 대화 내역에서 작업 추출

**수집 정보:**
- 무엇을 했는지
- 어떻게 했는지
- 왜 했는지
- 정량적 결과

---

## 단계 2: Write

수집된 정보를 문서화합니다.

**파일 생성:**
```
walkthrough/{오늘날짜}_walkthrough.md
예: walkthrough/2026-02-03_walkthrough.md
```

**작성 규칙:**
- 개조식, 간결하게
- 아이콘 없음
- 한글 사용
- 정량적 결과 필수

---

## 단계 3: Review

작성된 문서를 검토합니다.

**검토 항목:**
- 필수 항목 (무엇/어떻게/왜/결과) 포함 여부
- 형식 일관성
- 정량적 수치 구체성
- 상세도 충분성

**부족 시:**
- 수정 제안 제공
- 필요시 Write 단계 재실행

---

## 출력 템플릿

```markdown
- 총 변경 파일: N개
- 총 변경 라인: +X / -Y

---

## 작업 1: {작업명}

### 무엇을
- 대상:
- 범위:

### 어떻게
- 방법:
- 도구:

### 왜
- 배경:
- 목적:

### 결과
- 파일 N개 수정
- +X / -Y 라인
- 추가 수치...

---
```
