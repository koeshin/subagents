# Code Analyzer Skill

코드의 품질을 다양한 관점에서 분석합니다.

## 분석 항목

### 1. 보안 분석
- SQL Injection 취약점
- XSS (Cross-Site Scripting) 취약점
- 하드코딩된 시크릿/API 키
- 안전하지 않은 의존성

### 2. 복잡도 분석
- 함수별 Cyclomatic Complexity
- 파일별 라인 수
- 중첩 깊이 (Nesting Depth)

```
권장 기준:
- 함수당 복잡도: <= 15
- 함수당 라인 수: <= 50
- 중첩 깊이: <= 4
```

### 3. 중복 코드 탐지
- 10줄 이상 중복 블록 탐지
- 유사한 로직 패턴 식별
- 리팩토링 제안

### 4. 컨벤션 검사
- 네이밍 규칙 준수
- 파일 구조 일관성
- 코드 포매팅

---

## 사용법

분석하고 싶은 코드/파일/디렉토리를 지정하여 분석 요청:

```
"src/api 폴더의 코드 품질을 분석해줘"
"이 함수의 복잡도를 확인해줘"
"보안 취약점이 있는지 검사해줘"
```

---

## 출력 형식

```markdown
## 코드 분석 결과

### 요약
- 분석 파일 수: N개
- 발견된 이슈: N개
- 심각도: Critical N / Warning N / Info N

### 이슈 목록

| ID | 심각도 | 파일 | 라인 | 설명 |
|----|------|-----|-----|-----|
| 1 | 🔴 Critical | file.py | 42 | 하드코딩된 API 키 |
| 2 | 🟡 Warning | file.py | 88 | 함수 복잡도 과다 (18) |
| 3 | 🔵 Info | file.py | 120 | 네이밍 규칙 위반 |

### 권장 조치
1. [Critical] API 키를 환경 변수로 이동
2. [Warning] 함수를 더 작은 단위로 분리
```

---

## 심각도 기준

| 레벨 | 설명 | 즉시 조치 |
|-----|-----|---------|
| 🔴 Critical | 보안 위험, 기능 오류 | 필수 |
| 🟡 Warning | 품질 저하, 유지보수 어려움 | 권장 |
| 🔵 Info | 개선 가능 사항 | 선택 |
